引言

  • javascript中也是万物皆对象(这一点意外的像Java)
  • prototype仅仅在构造的时候起作用,确定了构造出来的对象的__proto__
  • prototype.constructor仅仅指向对象的构造函数
  • __proto__ the prototype chain中唯一的神，真正确定了原型链的关系
    • 但是要注意__proto__不是一个标准属性
    • 应该使用Object.getPrototypeOf和Object.setPrototypeOf替换(在生产环境中)
  • 一点辨析
    • Object构造函数 : Object
    • Object原型
    • Object实例
    • Function构造函数 : Function
    • Function原型
    • Function实例
  • __proto__ 隐式原型，prototype 显示原型，这个定义看着开心就好
  • 在js中原型一般是被’隐藏’的,无法被直接访问,需要通过实例.__proto__或者构造函数.prototype去访问

__proto__ (自己的所在的原型链)

  • 唯一的神，原型链中的chain的最终体现
  • 当对象访问属性时，会顺着__proto__去找
  • null是最顶层的原型
  • null --> object{} --> function() --> Object,Function

prototype(决定了构造出来的是对象的原型是什么)

  • 只有函数才有prototype属性(函数对象是没有的)
  • prototype决定了new出来的是一个什么东西

一分为二的constructor

函数.prototype.constructor(指向函数自己的套娃)

原型.constructor(指向创造自己的函数)

constructor在不被改变的情况下，指向该对象的构造函数

constructor包含在prototype中,如果不做任何特殊操作函数.prototype.constrotor === 函数

Warning

new

Function.prototype.apply and call的作用是

  • 函数1.apply(对象1,[参数列表])将对象1作为函数1中的this，并调用函数1

MDN explaination

new 关键字会进行如下的操作：

  1. 创建一个空的简单JavaScript对象；
  2. 链接该对象（即设置该对象的构造函数）到另一个对象 ；
  3. 将步骤1新创建的对象作为this的上下文 ；
  4. 如果该函数没有返回对象，则返回this。

当代码 new Foo(…) 执行时，会发生以下事情：

  1. 一个继承自 Foo.prototype 的新对象被创建。
  2. 使用指定的参数调用构造函数 Foo，并将 this 绑定到新创建的对象。new Foo 等同于 new Foo()，也就是没有指定参数列表，Foo 不带任何参数调用的情况。
  3. 由构造函数返回的对象就是 new 表达式的结果。如果构造函数没有显式返回一个对象，则使用步骤1创建的对象。（一般情况下，构造函数不返回值，但是用户可以选择主动返回对象，来覆盖正常的对象创建步骤

Warning

  • 如果自定义的构造函数在调用的时候没有加new，其中的this会指向全局window``
  • 然后 emmmmmm 属性就变成全局变量了

总览

贴上所有的属性和术语

CSS Properties(CSS属性)

  • flex-flow (shorthand for flex-direction,flex-wrap)(control Flex-Container)
    • flex-direction default: row (Change the Main Axis)
    • flex-wrap defualt: nowarp (sets whether flex items are forced onto one line or can wrap onto multiple lines)
  • flex (shorthand for flex-grow,flex-shrink,flex-basis)(control Flex-Item)
    • flex-grow default: 0
    • flex-shrink default: 1
    • flex-basis default: auto
  • order (change visual order and sorted by order)

Alignment Properties(对齐属性)(a sence of CSS Box Alignment)

  • justify-content (for all items on Main Axis)
  • align-content
  • align-items (for all items on Cross Axis)
  • align-self (for an items on Cross Axis)
  • Main used in Grid
    • gap
      • row-gap
      • column-gap
    • place-content(shorthand)
    • place-items(shorthand)

Glossary entries(术语表)

  • Flexbox
  • Flex Container
  • Flex Item
  • Main Axis
  • Cross Axis
  • Flex (just mean a shouthand)

Detail

Flex CSS Properties

  1. flex-wrap: 决定是否可以换行显示
    • nowrap不换行，超过则移除
    • wrap 换行
  2. flex
    • flex-basis: 指定了flex item在Main Axis上的初始大小
      • 比width or height有更高的优先级
    • flex-grow
    • flex-shrink
    • 不能是负值，值越大收缩越多(<number>)
  3. Warning:
    • flex-grow 是按照比例分配剩余空白区域，想要用这个属性作出平分效果可以试试flex-basis
    • flex-shrink 则是按照比例“缩水”超出width的flex-items

Alignment Properties(a sence of CSS Box Alignment)

  1. align-items and justify-content 是类似的
  2. align-content 对于单行的flex containner无效(flex-wrap:nowrap是一种)
    • 相当于将一行flex container作为一个item做对齐

一点总结

  • 在主轴上排列，在交叉轴上对齐
    • Main axis以排列为主，而Cross axis是可以花式对齐

奇技淫巧

  • work with margin auto
