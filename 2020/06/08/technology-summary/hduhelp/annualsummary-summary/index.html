心路历程与开发历程

根据GitHub的commit记录复盘

2020年5月25日，也是在灵光一闪后策划了一段时间，决定做一个“杭电学期报告”annualsummary，也算是第一个自己认可的项目。

也是在同一天，创建仓库，开工。
也是在同一天，第一次重构。

因为本来是打算自己处理从现有接口拿来的数据，但是该方案实际不可行。于是，叫来了FD Kevin做后端，而我来写前端。并在当天定下来json的基本结构，于是，正式开工。

那么，首先规划了一下项目结构(在重构是起到了很大的作用)，使用Swiper组件实现滑动效果，并开始着手写页面。这时的我还没有意识到，前方有两个大坑等着我。

到5月31日，终于是陆陆续续完成了除了end page(莫名其妙咕到了最后的end page)以外的所有page的制作，接入了cas登录，接入了API，看起来有模有样了，本以为基本可以完成了。总代码行数大概是4000+行(所有代码行数计算去除yarn-error-log)。

但是，此时的布局方案采用relative以及使用Swiper组件实现切换效果，却是两个大坑。

在测试中，relative + @media实现适配屏幕的方案频频出现问题，日常出现布局爆炸。以及，Swiper组件在IOS的Safari上出现了严重的性能问题，造成严重卡顿，效果堪比烤机工具。

此时，学长给出的意见是：建议重构，或者改用canvas

说实话，当晚心态有些爆炸。不过也还好，毕竟知道自己技术不行，还需要加强。

6月1日，快速自学了一下Grid+Flex，使用其重构了所有page(状态page的重构被鸽到了最后)，并且还原了原先的视觉效果。

现在回想起来挺意外的，没有试图去挽救relative布局的问题，而是果断重构。其实，在开始的时候，我就已经做好的会面积重构的准备，只是当时因为工作量太大，有一些逃避导致心态有一点不太好。

6月2日，试图在现有基础上优化Swiper在Safari上的性能。

也是挺意料之中的，性能优化以失败告终。

应该也是6月2日吧，也大概是这个时候，决定用animate.css来制作视觉效果。

6月3日，第一个commit就是remove Swiper，也是彻底放弃了这个库吧。使用animate.css重新实现了切换页面的视觉效果。

6月4日，引入hammer.js实现比较难自己实现的滑动效果，稍微踩了一下vue-hammer库(vue-hammer已经停止维护了，别用)和箭头函数没有预期this的坑。不过总体也是挺顺利的。

至此，主体开发完成，开发正式进入了debug阶段。5日，6日陆陆续续在使用层面上解决了overflow-scroll on Safari的问题，适配异形屏幕，电脑端提示(最后一个问题)等问题。并对主体做了一个clear。随着最后一次clear commit,annualsuammary的前端真正进入了第一个稳定版本。

技术栈

  • 使用 Vue 2.0 + Vuex,并没有使用Vue-router的必要
  • 一些依赖: hammer.js + animate.css + axios

踩坑实录

  • Swiper虽然是一个很大的滑动组件库，但是慎用，页面和特效过多时，会在safari产生严重的性能问题。
  • Vue.config.js中不能使用import需要使用require，并且不能使用alias，因为Vue.config.js是入口文件，并不会走webpack打包。
  • 永远不要使用relative作为主要布局手段
  • overflow-auto在safari上的特殊处理方式
  • CSS中的100%与100vw，z-index等等(CSS浑身上下都是坑)

奇技淫巧

  • 使用let that = this将this带入箭头函数
  • CSS variable可以在子CSS中更改并使用
  • devServer的使用
  • vue public directory，例如public/pic/1.JPG可以直接通过/pic/1.JPG调用
  • 用javascript获取页面宽度，并用页面宽度改写html font size

满意之处

  • 项目结构的可拓展性不错
  • 至少做出来了
  • 使用了Grid + Flex布局

一些遗憾

  • 因为版权方面的担忧，不能使用东方project的音乐作为背景音乐
  • 对于异形屏幕使用了只适配屏幕一边的方牺牲了最佳了的视觉体验
  • 不是真正的滑动效果
  • 心心念念的parallel X没有加进去
  • 后端不是自己写的
  • 背景效果没有用上canvas
  • 没有使用vscode
  • 返回首页按钮的delay实现没有用async实现(还是只会用promise)(其实promise也是一知半解)

收获了什么

  • 第一个拿得出手的项目
  • 论firefox MDN文档的重要性
  • 在项目的后段将Git GUI从source tree换成了tower
  • 顺手整了一个Hexo + Github Page
  • 大概确定了下一个项目的方向
