建站过程

  1. 本地建站
    1. 选取合适的目录
    2. 打开顺手的Shell，切换到该目录
    3. hexo init
  2. 配置_config.yml
    1. #Site 部分就按照文档里的介绍来配置就行
    2. #URL 配置详见踩坑实录
    3. #Deployment repo填上自己仓库的URL(3中的仓库)
      branch一般填master就行
  3. GitHub Repository

    新建一个repo取名为 你的github名字.github.io
    如果需要在repo的setting里打开GitHub page

  4. 部署(deploy)

    hexo clean 清除public中的已生成内容(可选命令)
    hexo generate 生成静态网站
    hexo deploy 部署，如果前面的设置正确，则会推送到GitHub的对应仓库
    之后就可以通过GitHub page的URL访问了

  5. CNAME

踩坑实录

  1. Github仓库的名字
    • 不要玩花的，就严格使用你的GitHub名字.github.io作为仓库名字，这样可以方便_config.yml #URL的配置
  2. _config.yml的#URL配置(可以用于解决CSS加载不出来的问题)
    • 这里文档建议好好看文档的URL配置说明
    • 如果前面一切按照建站过程配置，那么你的URL就是https://你的GitHub名字.github.io并且没有子目录，root就是你的将URL除了协议(https)的部分在抄一遍
  3. CNAME
    • 在setting里设置了CNAME之后，每次hexo deploy之后都会重置的解决方法
    • 在source下创建名字为CANME的file，在里面写上你添加了CNAME的域名
    • 之后每一次deploy就会自动的添加CNAME了
